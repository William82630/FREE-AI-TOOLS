Tool Testing Guide

This guide explains how to use the testing tools to verify that all your tools are working correctly.

Note: These testing tools are meant for development and maintenance purposes only. They should not be accessible to end users in production.

1. Basic Page Load Testing

The test_tools.py script checks if all tool pages load correctly by making HTTP requests to each page and verifying the response status.

How to Run the Page Load Test

  1. Make sure your Flask server is running locally
  2. Install required dependencies: 
    pip install requests colorama
  3. Run the test script: 
    python test_tools.py
  4. Review the results in the terminal

The script will test each tool page and report any failures. This is a quick way to check if any pages are completely broken.

2. JavaScript Error Tracking

The JavaScript error tracker (error-tracker.js) captures and logs JavaScript errors that occur when users interact with your tools.

How to Implement JavaScript Error Tracking

  1. Include the error tracker script in your base template: 
    <script src="{{ url_for('static', filename='js/error-tracker.js') }}"></script>
    Add this just before the closing </body> tag.
  2. Optionally, enable server-side error logging:
    • Import and register the error logger blueprint in your Flask app: 
      from error_logger import error_logger_bp
app.register_blueprint(error_logger_bp)
    • Edit static/js/error-tracker.js to enable server-side logging: 
      sendToServer: true,
serverEndpoint: '/log-error',

Once implemented, JavaScript errors will be logged to the console and optionally to a server-side log file.

3. Manual Testing Checklist

For more thorough testing, you can use this checklist to manually test critical functionality of each tool:

Tool Name: ______________________

□ Page loads correctly
□ UI elements display properly
□ Form inputs accept valid data
□ Form validation works correctly
□ Tool performs its primary function
□ Results display correctly
□ Error handling works properly
□ Responsive design works on mobile
□ Footer links work correctly
Important: After making significant changes to your codebase, it's recommended to run the automated tests and manually test the affected tools.

4. Troubleshooting Common Issues

If you encounter issues with your tools, check these common problems:

  • JavaScript Console Errors: Open your browser's developer tools (F12) and check the console for errors
  • Network Requests: Check the Network tab in developer tools to see if API requests are failing
  • Server Logs: Check your Flask server logs for any backend errors
  • Cross-Browser Issues: Test in multiple browsers (Chrome, Firefox, Safari, Edge)
  • Mobile Responsiveness: Use browser developer tools to test different screen sizes

By regularly using these testing tools and methods, you can ensure that your tools remain functional as you make changes to your codebase.
